1 Building Pencil2D on Linux {#build_linux}
2 ==========================
3 
4 These are instructions for building Pencil2D on Linux. If you are using Windows go [here](@ref build_windows), and macOS go [here](@ref build_macos). This guide is primarily targeted towards developers. If you just want to use the latest version you can just download one of our [nightly builds](https://www.pencil2d.org/download/#nightlybuild).
5 
6 This tutorial was made with Ubuntu Xenial Xerus (16.04) and Arch Linux in mind, however you should be able to adapt this guide to other versions or distributions if necessary.
7 
8 ## Installing Dependencies
9 
10 There are a few things that must be installed in order to build Pencil2D. In this section we will go over the installation of each of these components in detail. For Ubuntu we describe both graphical and command-line methods of installation, choose whichever one you feel more comfortable with (and if you don't know the difference, choose graphical). For Arch Linux we describe only the command-line method since that is what most Arch users are used to.
11 
12 ### %Qt
13 
14 Pencil2D relies on the %Qt application framework so you must install it before you can successfully build the program.
15 
16 #### Ubuntu
17 
18 ##### Graphical method
19 
20 - Go to the [%Qt Downloads](https://www.qt.io/download-open-source/) and download the <i>%Qt Online Installer for Linux</i>.
21 - Executing this file will start the %Qt installer application. If you can't open it right away, you may have to right click on it and go to *Properties*, then in the *Permissions* tab select *Allow executing file as program* and then try opening it again.
22 - Click Next. It will give you the option to log in with your %Qt developer account. You can skip this if you don't have an account and do not want to register.
23 - Next specify a location for %Qt, put it somewhere you can find it in case you ever need to navigate to the %Qt files manually.
24 - Next you can select the components you wish to install. At the very least you should have Desktop GCC selected under the latest %Qt version. Also make sure %Qt Creator under the Tools section is being installed (at the time of writing there is no option to uncheck this, but it's worth double checking!)
25 - Agree to the license and begin the installation. It will take a long time to download all of the files, so be patient. When the installation is complete, press Done and it will launch %Qt Creator for you.
26 
27 ##### Command-line method
28 
29 Pencil2D must be built with the %Qt 5 framework. To install %Qt 5, run this command:
30 
31  sudo apt install qt5-default qt5-qmake libqt5xmlpatterns5-dev libqt5svg5-dev qtmultimedia5-dev
32 
33 If you want to install %Qt Creator (recommended), then also run the following command:
34 
35  sudo apt install qtcreator
36 
37 #### Arch Linux
38 
39 Pencil uses version 5 of the %Qt framework. To install all required components of %Qt, run this command:
40 
41  sudo pacman -S --needed qt5-multimedia qt5-svg qt5-xmlpatterns
42 
43 For a more pleasant development experience, you might want to install %Qt Creator as well (recommended). To do so, run the following command:
44 
45  sudo pacman -S --needed qtcreator
46 
47 ### Make and GCC/Clang
48 
49 You will need GNU Make and either GCC or CLANG to build Pencil2D.
50 
51 #### Ubuntu
52 
53 These are usually installed by default, so you don't have to worry about them. If however you encounter issues, you can run the following commands.
54 
55 - GNU Make and GCC:
56 
57  sudo apt install make g++
58 
59 - GNU Make and Clang:
60 
61  sudo apt install make clang
62 
63 #### Arch Linux
64 
65 On most Arch systems, these are installed early on, but if your system does not have them yet, you can install them by running the following commands.
66 
67 - GNU Make and GCC:
68 
69  sudo pacman -S --needed make gcc
70 
71 - GNU Make and Clang:
72 
73  sudo pacman -S --needed make clang
74 
75 ## Building the application
76 
77 Now it's time to build the application. If you do not already have the Pencil2D source, please go over {Downloading Pencil2D Source} (coming soon) before continuing.
78 
79 ### With %Qt Creator (recommended)
80 
81 - Open up the %Qt Creator application, and from the menu bar select **File** and then **Open File or Project**. Navigate to Pencil2D's root git folder and open the *pencil2d.pro* file.
82 - Next you'll be asked to configure your kits for the project. Kits determine compilers, target environment, and various build settings among other things. The Desktop option should be the only one checked. Click Configure Project to complete the kit selection.
83 - Now all you have to do to build is click the plain **green arrow** in the bottom left corner of the window or press `Ctrl+r` keys. A small progress bar will show up on the bottom right and console output will appear in the bottom section.
84 - If everything goes well then the version of Pencil2D that you build will open up automatically and you're done!
85 
86 If there is an error, the issues tab will open up at the bottom and display the error messages. Try searching it on the [Pencil2D Issue tracker](https://github.com/pencil2d/pencil/issues) or create an issue If you can't find anything. Be sure to include as much detail as you can in your report!
87 
88 ### With QMake/GNU Make
89 
90 If you do not have or do not want to use %Qt Creator for some reason then you can follow this two step process. First you have to use QMake to let %Qt do its preprocessing and generate the Makefiles. Cd to the root git directory for Pencil2D. To avoid cluttering the source directories with generated files, weâ€™ll create a subdirectory named build for those. For the next step you will need to know the correct mkspec for your computer. Use the table below to find it.
91 
92 | Compiler | 32-bit | 64-bit |
93 | -------- | ------------ | ------------ |
94 | GCC | linux-g++-32 | linux-g++-64 |
95 | Clang | linux-clang | linux-clang |
96 
97 Substitute \<mkspec\> for the mkspec of your desired configuration and run the command below:
98 
99  mkdir build; pushd build; qmake -r -spec <mkspec> CONFIG+=debug ..; popd
100 
101 Next you have to use GNU Make to actually compile the source code. Run the command:
102 
103  make -C build
104 
105 You can then open Pencil2D by running this from the source directory:
106 
107  ./build/bin/pencil2d
108
